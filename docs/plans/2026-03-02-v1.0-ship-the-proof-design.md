**Date**: 2026-03-02
**Author**: Claude Opus 4.6 + Yann Abadie
**Status**: [experimental] — pending implementation

## Mission

Close ALL open P0-P2 issues, activate local neural embeddings (P3.5),
and prepare for marketplace submission. The goal is not a new feature —
it's the proof that every feature works, is documented, and is usable
by a developer who has never seen the project.

## What Changes

### 1. Last 2 P0 Fixes

**P0.11 — /budget endpoint session-scoped**
- Add `GET /budget/:session_id` to daemon API
- Returns session-local budget (tokens remaining, tokens consumed, latency budget)
- Keep `GET /budget` as global fallback

**P0.12 — db_path CLI flag consumed by daemon**
- `aletheia start --db-path ./custom.db` passes path to daemon
- Daemon reads `METAYGN_DB_PATH` env var OR `--db-path` CLI flag
- MemoryStore::new() accepts path parameter

### 2. P1 Quality Fixes

**P1.1 — daemon-contract.md regeneration**
- Rewrite entire document to reflect actual API surface
- Version field in health endpoint → read from Cargo.toml version
- Document all endpoints: /health, /hooks/*, /budget, /metrics, /sandbox/*, /forge/*, /mcp/*

**P1.5 — Neural embeddings for graph nodes (EmbeddingGemma-300M)**
- Activate `embeddings` feature flag with fastembed v5 crate
- Default model: `onnx-community/embeddinggemma-300m-ONNX` (q8 quantized)
- ~200MB download on first use, cached in `~/.claude/aletheia/models/`
- Auto-embed graph node content on insert (async, non-blocking)
- Cosine similarity search for `aletheia recall --query`
- Fallback: hash-based embeddings when feature disabled (existing behavior)

**P1.7 — Real Brier score in eval**
- Implement proper Brier score: `BS = (1/N) * Σ(confidence - outcome)²`
- Parse confidence from hook responses, outcome from session_outcomes
- Display calibration curve (bucketed: 0-20%, 20-40%, ..., 80-100%)
- Show both raw Brier score and reliability diagram in `aletheia eval`

**P1.8 — Document hooks/daemon split**
- Update architecture-notes.md with clear table:
  - Hooks with daemon endpoints (4): UserPromptSubmit, PreToolUse, PostToolUse, Stop
  - Hooks with TS-only logic (4): SessionStart, PostToolUseFailure, PreCompact, SessionEnd
- Explain why: latency budget (daemon call < 350ms), some hooks need no state

### 3. P2 Polish

- CI runs `pnpm build` and `pnpm typecheck`

**P2.5 — tool_input type alignment**

**P2.7 — Wire pruner.prune() into context**

- Add prune threshold config (default: 5 repeated errors)

**P2.8 — Clean orphan SessionContext fields**
